## đź“‹ Phase Workflows

Each phase follows this pattern:

1. **Context loading:** Read spec.yaml, project memory, prior artifacts
2. **Phase execution:** Generate requirements/design/code/tests/docs
3. **Artifact creation:** Write output to `.sdlc/workflow/artifacts/`
4. **PR creation:** Branch, commit, push, open GitHub PR
5. **State transition:** Mark phase as `awaiting_approval`
6. **Approval wait:** Human reviews on GitHub â†’ webhook triggers next phase

## đźŽŻ Phase-Specific Workflows

### Requirement Discovery (`sdlc-requirement`)

- Interview user about product goals, features, constraints
- Generate structured requirements document
- Output: `requirements.md` artifact
- Branch: `sdlc-<project>-requirements`

### Design (`sdlc-design`)

- Read requirements artifact
- Create architecture, component diagrams, data models
- Output: `design.md` artifact
- Branch: `sdlc-<project>-design`

### Planning (`sdlc-plan`)

- Break down design into implementation stories
- Estimate complexity, dependencies
- Output: `plan.md` with story list
- Branch: `sdlc-<project>-plan`

### Implementation (`sdlc-implement`)

- Execute current story (state machine tracks which one)
- Each story â†’ separate branch + PR
- Write code following existing patterns
- Branch: `sdlc-<project>-STORY-NNN`

### Testing (`sdlc-testing`)

- Write comprehensive tests (unit, integration, E2E)
- Capture evidence (screenshots for UI features)
- Output: Test files + evidence document
- Branch: `sdlc-<project>-testing`

### Documentation (`sdlc-docs`)

- Generate user docs, API references, setup guides
- Update README
- Branch: `sdlc-<project>-docs`

## đź’ˇ Best Practices

1. **Read state first:** Use `WorkflowState(project_dir)` to check current phase/story
2. **Single phase focus:** Complete one phase at a time; don't jump ahead
3. **Respect state machine:** Use `WorkflowState` methods, never edit `state.json` manually
4. **Feedback loop:** If phase fails approval, read `.sdlc/feedback/<phase>.md`
5. **Artifacts in .sdlc/:** Write all outputs to `.sdlc/workflow/artifacts/`, not /tmp

## đźš¨ Avoid

- Bypassing state machine (editing state.json directly)
- Skipping approval gates
- Deleting .sdlc/ directory
- Creating files outside project directory
- Re-introducing legacy patterns (e.g., manual agent session parsing)
